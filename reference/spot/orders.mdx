All order management methods are **private endpoints** — they require a valid API key and secret. These methods cover the full order lifecycle: placing single and batch orders, amending open orders, cancelling individual or all orders, and the Dead Man's Switch safety mechanism.

```typescript
import { SpotClient } from '@siebly/kraken-api';

const client = new SpotClient({
  apiKey: process.env.API_SPOT_KEY,
  apiSecret: process.env.API_SPOT_SECRET,
});
```

<Note>
  **API key permissions required:** Orders and trades — Create & modify orders. Use `getAssetPairs()` to look up trading pair constraints (`ordermin`, `costmin`, `tick_size`) before placing orders.
</Note>

## `generateNewOrderID()`

A utility helper that generates a unique 32-character hex string suitable for use as a `cl_ord_id`. Call this before `submitOrder()` to track your orders without depending on the Kraken-assigned `txid`.

**Signature**

```typescript
generateNewOrderID(): string
```

<CodeGroup>
```typescript Generate a Client Order ID
import { SpotClient } from '@siebly/kraken-api';

const client = new SpotClient({ /* ... */ });

const clOrdId = client.generateNewOrderID();
// e.g. 'a3f1c2d4e5b60789abcdef0123456789'

const order = await client.submitOrder({
  ordertype: 'limit',
  type: 'buy',
  pair: 'XBTUSD',
  volume: '0.001',
  price: '50000',
  cl_ord_id: clOrdId,
});
```
</CodeGroup>

## `submitOrder(params)`

Place a new order on the spot exchange. Supports all order types including market, limit, stop-loss, take-profit, and trailing stops.

**Signature**

```typescript
submitOrder(
  params: SpotSubmitOrderParams
): Promise<SpotAPISuccessResponse<SpotSubmitOrderResponse>>
```

**Required parameters**

<ParamField body="ordertype" type="string" required>
  Order type. One of:
  `market`, `limit`, `iceberg`, `stop-loss`, `take-profit`, `stop-loss-limit`, `take-profit-limit`, `trailing-stop`, `trailing-stop-limit`, `settle-position`.
</ParamField>

<ParamField body="type" type="'buy' | 'sell'" required>
  Order direction.
</ParamField>

<ParamField body="volume" type="string" required>
  Order quantity in base currency (as a string-encoded decimal).
</ParamField>

<ParamField body="pair" type="string" required>
  Asset pair to trade (e.g. `"XBTUSD"`).
</ParamField>

**Optional parameters**

<ParamField body="price" type="string">
  Limit price for `limit`, `stop-loss-limit`, `take-profit-limit` orders. Stop price for `stop-loss`, `take-profit`, `trailing-stop`, `trailing-stop-limit` orders.
</ParamField>

<ParamField body="price2" type="string">
  Secondary price for limit portion of `stop-loss-limit` and `take-profit-limit` orders, or trailing offset for trailing stops.
</ParamField>

<ParamField body="trigger" type="'index' | 'last'">
  Price signal to trigger stop orders. Defaults to `last`.
</ParamField>

<ParamField body="leverage" type="string">
  Leverage factor for margin orders (e.g. `"2:1"`).
</ParamField>

<ParamField body="reduce_only" type="boolean">
  If `true`, only reduces an existing position (no new position opened).
</ParamField>

<ParamField body="stptype" type="'cancel-newest' | 'cancel-oldest' | 'cancel-both'">
  Self-trade prevention type.
</ParamField>

<ParamField body="oflags" type="string">
  Comma-delimited order flags:
  - `post` — post-only order (maker only)
  - `fcib` — prefer fee in base currency
  - `fciq` — prefer fee in quote currency
  - `nompp` — no market price protection
  - `viqc` — volume in quote currency
</ParamField>

<ParamField body="timeinforce" type="'GTC' | 'IOC' | 'GTD' | 'FOK'">
  Time-in-force policy. `GTC` = Good-Til-Cancelled, `IOC` = Immediate-Or-Cancel, `GTD` = Good-Til-Date, `FOK` = Fill-Or-Kill.
</ParamField>

<ParamField body="starttm" type="string">
  Scheduled start time. `0` for now, `+<n>` for n seconds from now, or Unix timestamp.
</ParamField>

<ParamField body="expiretm" type="string">
  Expiration time. Same format as `starttm`.
</ParamField>

<ParamField body="deadline" type="string">
  RFC3339 timestamp after which the order should be rejected if not yet placed (max 60 seconds in the future).
</ParamField>

<ParamField body="cl_ord_id" type="string">
  Client-defined order ID (max 36 characters). Recommended — use `generateNewOrderID()` to create one.
</ParamField>

<ParamField body="userref" type="number">
  Integer user reference ID for grouping orders.
</ParamField>

<ParamField body="displayvol" type="string">
  Visible quantity for iceberg orders. Must be less than `volume`.
</ParamField>

<ParamField body="validate" type="boolean">
  If `true`, validates the order but does not submit it. Useful for parameter testing.
</ParamField>

<ParamField body="'close[ordertype]'" type="string">
  Closing order type to attach (conditional close). E.g. `"limit"`.
</ParamField>

<ParamField body="'close[price]'" type="string">
  Closing order price.
</ParamField>

<ParamField body="'close[price2]'" type="string">
  Closing order secondary price.
</ParamField>

<ParamField body="asset_class" type="'tokenized_asset'">
  Asset class of the pair.
</ParamField>

| Property | Value |
|----------|-------|
| **HTTP method** | `POST` |
| **Endpoint** | `0/private/AddOrder` |
| **Auth required** | Yes |

**Response shape** — `SpotSubmitOrderResponse`

| Field | Type | Description |
|-------|------|-------------|
| `descr.order` | `string` | Human-readable order description |
| `descr.close` | `string?` | Close order description (if attached) |
| `txid` | `string[]` | Array of transaction IDs assigned by Kraken |

## `cancelAllOrdersAfter(params)`

**Dead Man's Switch.** Sets a countdown timer that will automatically cancel all client orders if not reset before expiry. Send `timeout: 0` to disable the timer. Designed to protect against network disconnections or system failures.

**Signature**

```typescript
cancelAllOrdersAfter(params: { timeout: number }): Promise<
  SpotAPISuccessResponse<{
    currentTime: string;
    triggerTime: string;
  }>
>
```

<ParamField body="timeout" type="number" required>
  Number of seconds until all orders are cancelled. Set to `0` to disable the timer. Maximum: 86400 (24 hours).
</ParamField>

| Property | Value |
|----------|-------|
| **HTTP method** | `POST` |
| **Endpoint** | `0/private/CancelAllOrdersAfter` |
| **Auth required** | Yes |

**Response fields**

| Field | Type | Description |
|-------|------|-------------|
| `currentTime` | `string` | Server time when request was received |
| `triggerTime` | `string` | Scheduled cancellation time |

<CodeGroup>
```typescript Set 60-Second Timer
import { SpotClient } from '@siebly/kraken-api';

const client = new SpotClient({
  apiKey: process.env.API_SPOT_KEY,
  apiSecret: process.env.API_SPOT_SECRET,
});

// Start 60-second countdown
const dms = await client.cancelAllOrdersAfter({ timeout: 60 });
console.log('Will cancel at:', dms.result.triggerTime);

// Heartbeat loop — reset every 30s
setInterval(async () => {
  await client.cancelAllOrdersAfter({ timeout: 60 });
}, 30_000);

// Disable when shutting down cleanly
await client.cancelAllOrdersAfter({ timeout: 0 });
```
</CodeGroup>

<Note>
  You must **continuously reset** the timer before it expires. The timer resets each time you call `cancelAllOrdersAfter()` with a non-zero timeout. If your process crashes, the timer will fire and cancel all orders.
</Note>

## `submitBatchOrders(params)`

Submit a collection of orders (minimum 2, maximum 15) for a single trading pair. Validation is applied to the entire batch before any orders are placed — if one order fails validation, the whole batch is rejected.

**Signature**

```typescript
submitBatchOrders(params: SpotSubmitOrderBatchParams): Promise<
  SpotAPISuccessResponse<{ orders: SpotBatchOrderResult[] }>
>
```

<ParamField body="pair" type="string" required>
  Asset pair for all orders in the batch. All orders must share the same pair.
</ParamField>

<ParamField body="orders" type="SpotBatchOrderItem[]" required>
  Array of 2–15 order objects. Each item supports the same fields as `submitOrder()` except `pair`.
</ParamField>

<ParamField body="validate" type="boolean">
  If `true`, validates the batch but does not submit. Useful for parameter testing.
</ParamField>

<ParamField body="deadline" type="string">
  RFC3339 deadline timestamp for the batch.
</ParamField>

<ParamField body="asset_class" type="'tokenized_asset'">
  Asset class filter.
</ParamField>

<ParamField body="broker" type="string">
  Broker identifier (if applicable).
</ParamField>

| Property | Value |
|----------|-------|
| **HTTP method** | `POST` |
| **Endpoint** | `0/private/AddOrderBatch` |
| **Auth required** | Yes |

**Response** — `{ orders: SpotBatchOrderResult[] }`

Each `SpotBatchOrderResult` contains `txid?`, `descr?`, and `error?` (if that specific order failed).

<CodeGroup>
```typescript Submit a Batch
import { SpotClient } from '@siebly/kraken-api';

const client = new SpotClient({
  apiKey: process.env.API_SPOT_KEY,
  apiSecret: process.env.API_SPOT_SECRET,
});

const batch = await client.submitBatchOrders({
  pair: 'XBTUSD',
  orders: [
    {
      ordertype: 'limit',
      type: 'buy',
      volume: '0.0001',
      price: '45000.00',
      timeinforce: 'GTC',
      cl_ord_id: client.generateNewOrderID(),
    },
    {
      ordertype: 'limit',
      type: 'buy',
      volume: '0.0001',
      price: '44000.00',
      timeinforce: 'GTC',
      cl_ord_id: client.generateNewOrderID(),
    },
    {
      ordertype: 'limit',
      type: 'sell',
      volume: '0.0001',
      price: '55000.00',
      timeinforce: 'GTC',
      cl_ord_id: client.generateNewOrderID(),
    },
  ],
});

batch.result.orders.forEach((o, i) => {
  if (o.error) {
    console.error(`Order ${i} failed:`, o.error);
  } else {
    console.log(`Order ${i} placed:`, o.txid);
  }
});
```

## Full Order Lifecycle Example

The following example shows a complete order cycle — place a limit order, verify it appears in open orders, then cancel it.

<CodeGroup>
```typescript Order Lifecycle
import { SpotClient } from '@siebly/kraken-api';

const client = new SpotClient({
  apiKey: process.env.API_SPOT_KEY,
  apiSecret: process.env.API_SPOT_SECRET,
});

// 1. Generate a unique client order ID
const clOrdId = client.generateNewOrderID();

// 2. Place a limit order (well below market to avoid fill)
const submitted = await client.submitOrder({
  ordertype: 'limit',
  type: 'buy',
  volume: '0.001',
  pair: 'XBTUSD',
  price: '10000',
  timeinforce: 'GTC',
  cl_ord_id: clOrdId,
});
const txid = submitted.result.txid[0];
console.log('Order placed, txid:', txid);

// 3. Confirm it appears in open orders
const openOrders = await client.getOpenOrders({ cl_ord_id: clOrdId });
console.log('Open order status:', openOrders.result.open[txid]?.status);

// 4. Cancel the order
const cancelResult = await client.cancelOrder({ txid });
console.log('Cancelled:', cancelResult.result.count, 'order(s)');
```
